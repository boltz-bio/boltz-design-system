# Boltz Design System

Components, tokens, and Storybook for [Boltz Bio](https://boltz.bio). Built and maintained by [G-W Studios](https://github.com/G-W-Studios).

## Quickstart

```bash
pnpm install
pnpm dev:storybook
# → http://localhost:6006
```

## Structure

```
boltz-design-system/
├── packages/
│   ├── tokens/           # Design tokens (color, type, spacing, radius, motion)
│   └── ui/               # React components — Radix + Tailwind
├── apps/
│   └── storybook/        # Storybook (component docs + sections + screens)
├── DESIGN.md             # Visual spec — generated by Claude Design
├── KNOWLEDGE.md          # Product domain — what Boltz is, audiences, scope
├── COPY.md               # Brand voice — tone, copy patterns
├── MAPPING.md            # Component naming: DESIGN.md ↔ Figma ↔ code
└── reference/            # Reference materials (Figma exports, source HTML)
```

## Source of truth

| Concern | Source |
|---|---|
| Visual rules | `DESIGN.md` |
| Product domain | `KNOWLEDGE.md` |
| Brand voice / copy | `COPY.md` |
| Token values | `packages/tokens/src/tokens.ts` (synced with Figma) |
| Figma file | [Boltz web 2.0](https://www.figma.com/design/LvTmQRNQ2FZ6GcrSpwuvgl/Boltz-web-2.0) |

## Add a component

```bash
claude /g-w-component <Name>
```

The `g-w-component` skill reads `DESIGN.md` from project root, generates the component + Storybook story, and adds the export.

## Hard rules

1. No hex/rgb/hsl in component code — use tokens via Tailwind classes
2. Stabil Grotesk Regular 400 only — no other weights
3. `rounded-full` (9999px) on all buttons, chips, single-line inputs
4. One black CTA per viewport band
5. Eyebrow label "• Section name" before major section headings
6. No drop shadows (exception: `ex-toast`)
