# Boltz Design System

Components, tokens, and patterns for the Boltz Bio brand. Built and maintained by **G-W Studios**.

---

## Brand essence

> Boltz is a frontier AI research company for biomolecular design.
> Three strategic messages in strict priority:
> **1. Frontier Capabilities** — state-of-the-art models, research credibility.
> **2. Flexibility / Modularity** — engineering quality, "the foundation".
> **3. Popularity / Distribution** — community scale, ecosystem.

Visual signature: warmth (Tierra-50 canvas, not white). Black is the only action color. Sage is atmospheric — never a button fill. **Pill geometry** (`rounded-full`) on everything interactive. **No drop shadows.** Stabil Grotesk Regular 400 — never another weight.

## What's in here

### 01 — Foundations
- **Colors** — Tierra, Sage, Blue, Neutrals, Status (full token swatch)
- **Typography** — Stabil Grotesk type scale, 6 tokens
- **Tokens** — Spacing, Radius, Motion

### 02 — Components (v0.0.1, growing weekly)
- `Button` — Primary, Secondary, OnDark with ↗ suffix variants
- `EyebrowLabel` — Outlined pill with Iconoir leaf icon, "• Section name" pattern
- `NavCta` — Persistent nav CTA "Get early access ↗"
- `IconContainer` — 40×40 wrapper for inline icons, light + dark
- `Card` — Light, Dark, Blue variants

### 04 — Sections
- `HeroBand` — Composition proof: eyebrow + heading-lg + body-lg + buttons

## Source of truth

Every visual rule lives in **`DESIGN.md`** at the repo root. Every product rule lives in **`KNOWLEDGE.md`**. Every voice rule lives in **`COPY.md`**.

When adding code, AI agents must read these files first. The `g-w-component` Claude skill enforces this.

## Hard rules — NEVER violate

1. No hex/rgb/hsl in component code — only tokens
2. Stabil Grotesk Regular 400 only — no other weights
3. `rounded-full` (9999px) on all buttons, chips, single-line inputs
4. One black CTA per viewport band
5. Eyebrow label "• Section name" before major section headings
6. No drop shadows (single exception: `ex-toast`)
7. ↗ suffix on all CTA text (typographic or icon-button variant)

## Add a component

In the repo:

```bash
claude /g-w-component <Name>
```

The skill reads DESIGN.md, builds the component, writes the story, and updates the export. AI is not allowed to invent components that don't appear in DESIGN.md.
