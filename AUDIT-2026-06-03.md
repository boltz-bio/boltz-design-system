# Audit 2026-06-03 — Figma vs DESIGN.md

When `tokens.ts` and `tailwind.config.cjs` deviate from `DESIGN.md`, this file logs why.

## Findings (initial)

### Typography — Figma wins

| Token | DESIGN.md | Figma | Used in code |
|---|---|---|---|
| heading-lg | 56px / `-1.0px` | **62px** / `-4%` | `62px / -0.04em` |
| heading-md | 32px / `-0.5px` | 32px / `-2%` | `32px / -0.02em` |
| heading-sm | 24px / `-0.3px` | 24px / `-1%` | `24px / -0.01em` |
| body-lg | 20px / normal | 20px / `-2%` | `20px / -0.02em` |
| body-md | 18px / normal | 18px / `-1%` | `18px / -0.01em` |
| body-sm | 15px / normal | 15px / `-1%` | `15px / -0.01em` |

Rationale: Figma is the live design source. DESIGN.md was likely written with rounded pixel values for human readability; Figma uses percent tracking which converts to em naturally.

### Colors — 100% match

17 Figma variables match `tokens.json` from colors.zip exports.

### Components published in Figma

Only 3 components currently published:
- `Primary button` (3 variants: Filled, outlined, white)
- `Tag` (= EyebrowLabel)
- `Nav button` (= NavCta)

The remaining ~40 components in DESIGN.md are not yet published in Figma. We build them from DESIGN.md spec; designer should publish from Figma when ready.

### EyebrowLabel icon

DESIGN.md says literal "•" bullet. Figma Tag component shows leaf icon (Iconoir). Code defaults to leaf, exposes bullet as opt-in.

### ↗ suffix rendering

Two visual patterns in Figma: typographic arrow inside pill (Filled button) vs circular outlined icon-button next to pill (some variants). Code exposes both via `suffix` prop.
